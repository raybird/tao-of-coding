# oh-my-opencode-slim 專案建立精神與架構分析

本文件旨在記錄 `alvinunreal/oh-my-opencode-slim` 專案的核心精神、架構設計及其相對於原始版本的優化重點。

## 1. 核心精神：從混亂中建立秩序 (Order from Chaos)

`oh-my-opencode-slim` 的核心哲學是將複雜的軟體開發過程，透過多個具備明確職責的 AI 代理（Agents）進行拆解與轉化。它不只是將 AI 視為聊天機器人，而是將其視為一個具備嚴謹分工的「開發角色體系」（Pantheon），透過協作來處理大規模且複雜的程式碼庫。

## 2. 角色分工：六大開發角色 (The Pantheon)

專案將代理分為六種特定角色，各司其職：

| 角色 (代理) | 稱號 | 核心職責 |
| :--- | :--- | :--- |
| **Orchestrator** | 秩序的體現 | 頂層策略協調運作，負責將任務細分並委派給合適的代理。 |
| **Explorer** | 永恆的流浪者 | 負責程式碼庫的偵查與掃描，理解檔案結構、關聯性與程式碼模式。 |
| **Oracle** | 路徑守護者 | 戰略顧問，提供架構設計決策指導、複雜 Bug 的邏輯分析與建議。 |
| **Librarian** | 知識編織者 | 外部資訊與技術文件檢索，將原始資料轉化為可執行的深度理解。 |
| **Designer** | 美學守護者 | 專注於前端與 UI/UX 的實作，確保視覺美感與互動邏輯符合現代化標準。 |
| **Fixer** | 最後的建築師 | 高效率的執行者，負責將策略與設計快速轉化為實際的程式碼變更。 |

## 3. 「Slim」版本的關鍵改進

此版本（Slim）是基於 `oh-my-opencode` 的優化分叉版，主要針對以下痛點進行改良：

- **顯著降低 Token 消耗**：透過精簡與微調 Prompt (System Instruction)，過濾掉冗餘資訊，讓開發成本大幅下降。
- **異步任務處理 (Fire-and-forget)**：最新版本引入了事件驅動的任務機制，使代理在執行後能立即釋放資源，提升系統反應速度。
- **模型混合策略 (Model Agnostic)**：支援根據不同代理的特性分配不同的 LLM 模型（如 Gemini, Claude, OpenAI, Cerebras 等），平衡效能與預算。
- **開發效能優化**：基於 Bun 運行環境與 TypeScript，並深度整合 MCP (Model Context Protocol) 支援。
